Faz 1 — İlk Keşif
1
Dosya türünü belirle
file <binary>
ELF mi? PE mi? Script mi? Packed mi? 32-bit mi 64-bit mi? — Her zaman buradan başla.
↓
2
String'leri tara
strings <binary> | grep -iE "flag|pass|key|correct"
Kolay challenge'ların ~%30'u burada çözülür. Flag görünürse bitti.
↓
3
Packing kontrol et
strings <binary> | grep UPX
Packed ise → upx -d binary -o unpacked → Adım 1'den başa dön.
↓
4
Kütüphane çağrılarını izle
ltrace ./binary | grep -i cmp
strcmp/memcmp yakalanır — binary'nin neyle karşılaştırdığı görünür.
Faz 2 — Statik Analiz
5
Ghidra — başarı string'ini bul
Window → Defined Strings → "Correct" / "Congratulations" ara
Çift tıkla → referans lokasyonuna atla → üstündeki koşulu oku.
↓
6
Doğrulama fonksiyon türünü belirle
(decompile edilmiş C çıktısını oku)
Direkt strcmp? → string al. XOR? → geri al. MD5? → ileri çalış. Çok aşamalı? → hepsini karşıla.
↓
7
Cevabı yeniden oluştur
python3 solve.py
Karmaşık yeniden yapılandırma için Python scripti yaz (indis eşleştirme, keygen, XOR).
Faz 3 — Dinamik Analiz
8
GDB / x64dbg — statik yetmediyse
gdb ./binary → b strcmp → run → x/s $rdi → x/s $rsi
Karşılaştırmadaki register'ları incele. $rdi = girdimiz, $rsi = beklenen değer.
↓
9
Binary ile doğrula
./binary → cevabı gir → ✅ Congratulations!
Asla binary'de test etmeden gönderme.
↓
10
Write-up yaz
docs/research/challenge-0N-name/writeup.md + research-notes.md
Komutlar + bulgular + flag + teknik kataloğu. Sadece write-up'tan yeniden çözülebilir olmalı.
